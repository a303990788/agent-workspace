# 共享上下文

## 项目

用于协调多个 agent 的在线联合工作区。

## 目标

提供一个稳定、可通过 URL 访问的共享事实来源，让多个 agent 可以读取和更新同一份上下文，而不依赖本地浏览器登录态或私有文档会话。

## 当前建议

使用 GitHub 仓库作为主要协作层：

- Markdown 用于人类可读的上下文。
- JSON 用于机器可读状态。
- Pull Request 或 commit 用于保留写入历史。
- GitHub raw URL 用于 agent 稳定读取。

## 非目标

- 不依赖飞书/Lark 浏览器会话。
- 不要求访问某台本地机器。
- 不依赖隐藏的浏览器登录态。

## 待确认问题

- 仓库 owner 和仓库名。
- 使用公开仓库还是私有仓库。
- 哪些 agent 需要写入权限。
- 任务更新采用直接 commit、Pull Request，还是 API 写入。
